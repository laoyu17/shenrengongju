## 1. 目标与范围

本文定义“研究闭环”在当前项目中的**机器可判定标准**，用于统一评审口径，避免“已实现/增强中”语义漂移。

- 评审入口：`rtos-sim run ... --audit-out <path>` 产出的 `compliance_profiles`
- 判定对象：协议一致性、资源终止路径、死锁与证明辅助轨迹
- 非目标：性能优化（本轮明确不作为达标条件）

## 2. 合规画像定义

### 2.1 engineering_v1（工程交付基线）
- 必过检查：
  - `resource_release_balance`
  - `abort_cancel_release_visibility`
  - `resource_partial_hold_on_block`
  - `wait_for_deadlock`
- 用途：工程回归、CI 快速门禁

### 2.2 research_v1（研究复现基线）
- 必过检查：
  - `resource_release_balance`
  - `abort_cancel_release_visibility`
  - `pcp_priority_domain_alignment`
  - `pcp_ceiling_numeric_domain`
  - `resource_partial_hold_on_block`
  - `pip_priority_chain_consistency`
  - `pcp_ceiling_transition_consistency`
  - `wait_for_deadlock`
  - `pip_owner_hold_consistency`
  - `time_deterministic_ready_consistency`
- 用途：论文/报告级实验的语义闭环判定

### 2.3 判定规则
- `compliance_profiles.profiles.<profile>.status == "pass"`：画像通过
- `failed_checks` 非空：画像失败，必须给出失败样例与修复说明
- `missing_checks` 非空：视为失败（禁止“规则未执行”绕过）
- `inspect-model` 关系画像允许 `pass/warn/fail` 分级；建议在 CI/脚本场景使用 `--strict-on-fail` 将 `warn/fail` 转为非 0 退出码
- `segment_core_binding_coverage` 对“迁移导向且无资源约束”的 `unbound` 仅做 advisory 统计（`advisory_unbound_segment_count`），不单独降级画像状态
- `research-case-summary` 中每个 case 需满足：
  - `expected_status == actual_status`
  - `missing_expected_checks` 为空
  - `unexpected_actual_checks` 为空（不允许多余失败项被忽略）

## 3. 执行与取证流程

1. 模型校验
   - `rtos-sim validate -c <config>`
2. 运行并导出审计
   - `rtos-sim run -c <config> --audit-out artifacts/audit.json --metrics-out artifacts/metrics.json --events-out artifacts/events.jsonl`
3. 导出模型关系
   - `rtos-sim inspect-model -c <config> --out-json artifacts/model_relations.json --out-csv artifacts/model_relations.csv`
4. 固化质量快照（测试/覆盖率）
   - `python scripts/quality_snapshot.py --output artifacts/quality/quality-snapshot.json --coverage-json artifacts/quality/coverage.json`
5. 执行研究反例基准集（fail/fix 配对）
   - `python scripts/research_case_suite.py --cases examples/research_counterexamples.json --out-json artifacts/research/research-case-summary.json --out-csv artifacts/research/research-case-summary.csv --audit-dir artifacts/research/case-audits`
6. 生成研究评审模板报告
   - `python scripts/research_report.py --audit artifacts/research/audit.json --relations artifacts/model_relations.json --quality artifacts/quality/quality-snapshot.json --out-markdown artifacts/research/research-report.md --out-csv artifacts/research/research-summary.csv --out-json artifacts/research/research-report.json`
   - 报告中需同时检查 `failed_check_details` 与 `non_audit_fail_details`
7. 评审结论
   - `research_v1.status`、`engineering_v1.status`
   - `model_relations.status`
   - `quality_snapshot.pytest.passed` 与 `quality_snapshot.coverage.line_rate`
   - `research-case-summary.status`（反例基准集匹配率）
   - `research-report.failed_check_details[*].issue_count/sample_count/sample_event_ids`
   - `research-report.non_audit_fail_details[*].source/status/failed_rules`
   - `audit.time_deterministic_proof_assets`（`max_ready_lag` / `max_phase_jitter` / `issue_samples`）

## 6. 后续迭代建议

- 增加 research_v1 反例基准集（多锁链、PCP 天花板边界、混合到达过程）
- 将 `research_v1` 判定接入 CI 非阻断报告；建议满足“连续 7 天 `research_v1=pass` 且 `unexpected_actual_checks` 全为空”后再评估升级为阻断门禁
- 在 CI Step Summary 中显式展示 `failed_rules`、`non_audit_fail_details` 与画像状态告警，降低“非阻断任务被忽略”风险
- 补充模板化审计报告（面向评审/论文附录直接引用）
