# 术语与数据模型草案

## 1. 术语表（节选）
- **Task**：任务集合中的一个任务，包含若干子任务。
- **Subtask**：任务 DAG 中的节点，具有前驱/后继关系。
- **Segment**：子任务内的顺序分段执行单元。
- **WCET/ACET**：最坏/平均执行时间。
- **Hyper-Period**：时间确定性任务的超周期。
- **Resource Binding**：共享资源固定在指定处理器核上访问。

## 2. 数据模型（草案）

### 2.1 ProcessorType / Core
- ProcessorType: id, name, core_count, speed_factor
- Core: id, type_id, speed_factor

### 2.2 Resource
- id, name, bound_core_id, protocol

### 2.3 TaskGraph
- id, name, task_type, period, deadline, arrival, abort_on_miss
- subtasks: list[Subtask]

### 2.4 Subtask (DAG Node)
- id, task_id
- predecessors: list[subtask_id]
- successors: list[subtask_id]
- segments: list[Segment]

### 2.5 Segment
- id, subtask_id, index
- wcet, acet
- required_resources: list[resource_id]
- mapping_hint: core_id | None

## 3. 事件模型（草案）
- JobReleased
- SegmentReady
- SegmentStart
- SegmentEnd
- ResourceAcquire
- ResourceRelease
- Preempt
- Migrate
- DeadlineMiss
- JobComplete

## 4. 约束关系（摘要）
- DAG 依赖：所有前驱完成后，子任务可就绪
- 分段顺序：第 k 段完成后，第 k+1 段可就绪
- 资源绑定：资源访问必须在绑定核上执行
- 实时性：动态实时任务需满足截止期约束
