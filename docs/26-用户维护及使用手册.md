# 用户维护及使用手册（增量）

## 1. CLI 新能力

### 1.1 计划指纹

`plan-static` 输出 JSON 新增 `spec_fingerprint`，用于标识模型语义。

### 1.2 严格计划匹配

- `analyze-wcrt --strict-plan-match`
- `export-os-config --strict-plan-match --config <path> --plan-json <path>`

当计划文件与配置指纹不一致时，命令返回码为 2。

### 1.3 strict 默认脚手架模板（推荐）

新增脚本：`review/scripts/strict_plan_pipeline.sh`（默认强制 strict）。

```bash
bash review/scripts/strict_plan_pipeline.sh examples/at06_time_deterministic.yaml
```

可选输出目录：

```bash
bash review/scripts/strict_plan_pipeline.sh examples/at06_time_deterministic.yaml review/runtime/i2/strict_plan_pipeline
```

脚本固定流程：
1. `plan-static`
2. `analyze-wcrt --strict-plan-match`
3. `export-os-config --strict-plan-match --config <path> --plan-json <path>`

失败诊断入口：
- `review/runtime/i2/strict_plan_pipeline/results.tsv`
- `review/runtime/i2/strict_plan_pipeline/logs/*.log`

CI 门禁：
- `.github/workflows/ci.yml` 新增 `strict_plan_gate`（独立非性能门禁，固定运行 strict 模板）。

### 1.4 双口径可调度率

`benchmark-sched-rate` 输出：

- 严格：`candidate_only_schedulable_rate`、`candidate_only_uplift`
- 兼容：`best_candidate_schedulable_rate`、`uplift`

## 2. UI Planning 面板

位置：左侧编辑区 `Planning` 页签。

功能：

1. 配置 planner / task scope / LP objective / WCRT 参数。
2. 一键执行 `Plan Static`、`Analyze WCRT`、`Export OS Config`。
3. 通过 `Random Task Generator` 生成可复现任务集。
4. 查看 Schedule/WCRT/OS Config 三张结果表与 Planning Logs。

## 3. 状态流展示

右侧遥测区新增状态流视图，展示：

- Released
- Ready
- Executing
- Blocked

## 4. 常见问题

1. 严格匹配失败：优先检查 `plan.json` 是否由当前 `--config` 重新生成。
2. `candidate_only_uplift` 低于 `uplift`：这是预期行为，说明兼容口径包含了 baseline 可调度样本。
