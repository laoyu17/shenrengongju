# 高标准论文图件规划（混合模式）

## 0. 文档控制
- 版本：v1.0
- 日期：2026-03-03
- 适用范围：`project/` 仓库论文图件生产与投稿排版
- 原则：仅建设离线图件资产流水线，不改变仿真系统产品能力

## 1. 目标与边界

### 1.1 目标
- 为期刊/顶会投稿提供可复现、可追溯、风格统一的图件生产流程。
- 将“结果可信度图件”与“封面感概念图”分离治理，避免混淆证据与视觉增强。

### 1.2 边界
- 不修改 `rtos_sim/core|model|cli|ui` 功能路径。
- 不把“出图能力”并入产品 UI/CLI 对外能力。
- 图件生产仅通过离线脚本与产物目录执行。

## 2. 当前落地状态
- 数据构建：`scripts/paper_assets/build_dataset.py`
- 主文图生成：`scripts/paper_assets/plot_main_figures.py`
- 附录图生成：`scripts/paper_assets/plot_appendix_figures.py`
- 风格统一：`scripts/paper_assets/style.py`
- 清单导出：`scripts/paper_assets/export_manifest.py`
- 概念图 Prompt：`scripts/paper_assets/prompt_nano_banana.md`
- 已生成图件：`artifacts/paper_figures/main/`（8 张） + `artifacts/paper_figures/appendix/`（6 张）
- UML 资产口径：`docs/uml-src/*.puml` 为权威源，`png/svg` 仅为可再生产物

## 3. 混合模式执行规范

### 3.1 直出图（必须本地脚本生成）
- 结果图：性能、置信区间、可复现治理指标。
- 消融图：资源协议/调度策略/规则覆盖差异。
- 机制图：架构证据链、fail-fix 时间线、合规画像。
- 要求：数据来源可回溯到 `artifacts/paper_data/*.csv` 与 `manifest`。

### 3.2 外部绘图（仅概念图）
- 用途：Graphical Abstract、封面感视觉摘要。
- 要求：
  - 不承载定量结论，不替代结果图。
  - 标签与术语与主文一致（`research_v1`、`engineering_v1`、`EventBus` 等）。
  - 使用 `scripts/paper_assets/prompt_nano_banana.md` 中的 master/variant prompt 与 negative prompt。

## 4. 图件矩阵（投稿建议）
- 主文 Fig.1-3：机制与证据链（架构、fail-fix、规则闭环）。
- 主文 Fig.4-6：合规画像与消融分析（profiles、proof assets、resource policy）。
- 主文 Fig.7-8：规模性与可复现治理（scalability CI、governance）。
- 附录 App.A1-A6：UI 工作流、到达分布、ETM 对比、事件组成、关系矩阵、文档一致性状态。
- 概念图 GA1：Nano Banana Pro 生成，置于首页或补充材料封面。

## 5. 质量门禁（投稿前必须全部通过）
- `python -m pytest --maxfail=1`
- `python scripts/check_doc_baseline_consistency.py --snapshot artifacts/quality/quality-snapshot.json --docs-root docs`
- `python scripts/paper_assets/build_dataset.py --regenerate`
- `python scripts/paper_assets/plot_main_figures.py`
- `python scripts/paper_assets/plot_appendix_figures.py`
- `python scripts/paper_assets/export_manifest.py`

## 6. 风险与规避
- 风险：概念图视觉风格偏离学术调性。
  - 规避：优先使用 conservative variant，并执行 half-size 可读性检查。
- 风险：图件与代码快照不一致。
  - 规避：每次投稿打包前强制 `--regenerate` 并更新 manifest 哈希。
- 风险：误把概念图作为证据图引用。
  - 规避：在图注中明确“Graphical Abstract（non-quantitative）”。
