## 4. 验收口径（冻结）
- 指标口径：保留兼容口径 `best_candidate_schedulable_rate/uplift`，新增严格口径 `candidate_only_schedulable_rate/candidate_only_uplift`。
- case 判定口径：静态规划 `feasible=true` 且 `WCRT feasible=true` 才计入“可调度”。
- 提升口径：`uplift = (candidate - baseline) / baseline`，其中严格门禁采用 `candidate_only_uplift`。
- 通过阈值：`candidate_only_uplift >= 0.30`。
- 统计口径：固定 seed、固定任务集生成器、分层汇总后给出总体结果。
- 兼容口径：旧 CLI 与旧配置在不启用新字段时行为不变。

## 5. 风险与控制（对齐计划摘要）
- LP 求解耗时风险：按规模分层 + 求解超时 + 启发式降级并保留证据。
- 一次性改动回归面大：规划器与执行器分层隔离，接口契约测试前置。
- 30% 目标场景差异：按负载分层统计并冻结聚合方式，避免临时改口径。
