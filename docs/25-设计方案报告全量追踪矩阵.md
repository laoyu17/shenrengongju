## 4. 验收口径（冻结）
- 指标口径：唯一验收门禁固定为 `candidate_only_uplift`；兼容口径 `best_candidate_schedulable_rate/uplift` 仅保留展示。
- case 判定口径：静态规划 `feasible=true` 且 `WCRT feasible=true` 才计入“可调度”。
- 提升口径：验收结论固定使用 `candidate_only_uplift = (candidate_only_rate - baseline_rate) / baseline_rate`。
- 通过阈值：`candidate_only_uplift >= 0.30`。
- 统计口径：固定 seed、固定任务集生成器、分层汇总后给出总体结果。
- 兼容口径：旧 CLI 与旧配置在不启用新字段时行为不变。

## 5. 风险与控制（对齐计划摘要）
- LP 求解耗时风险：按规模分层 + 求解超时 + 启发式降级并保留证据。
- 一次性改动回归面大：规划器与执行器分层隔离，接口契约测试前置。
- 30% 目标场景差异：按负载分层统计并冻结聚合方式，验收结论唯一使用 `candidate_only_uplift`，避免临时改口径。
- UI 研究化范围膨胀风险：本轮仅交付 Compare 报告/导出层与 Research Report 最小闭环，不将 DAG 深交互的产品化收口混入当前验收。
